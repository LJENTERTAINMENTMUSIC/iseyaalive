1. CORE PHILOSOPHY & RUNTIME MECHANICS
--------------------------------------------------------------------------------
IṢẸ́YÁÁ (Integrated Sports, Events, Youth, Arts & Attractions) is a government-
owned "Super-App" designed as a unified economic engine for Ogun State.

RUNTIME ARCHITECTURE: Zero-Build Browser-Native ESM
- Core Engine: React 19 (Transient state with persistent ledger simulation).
- Transpilation: Babel Standalone (v7) processing .tsx files at runtime.
- Package Resolution: Import Maps via esm.sh CDN.
- Secret Management: Credentials restricted to CI/CD environment variables.
- Domain Node: iseyaa.com anchored via GitHub Pages infrastructure.

2. SYSTEM LOGIC FLOW
--------------------------------------------------------------------------------
Phase 1: Entry & Authorization
- LandingPage.tsx acts as the primary gateway.
- AuthModal.tsx provides Role-Based Access Control (RBAC). 
- Access Tiering: Public User, Merchant Vendor, Govt Official, System Admin.

Phase 2: Modular Command Hub
- Dashboard.tsx: Central telemetry and KPI oversight.
- App.tsx: Core state-driven router managing sector modules.
- Layout.tsx: Dynamic sidebar filtering based on biometric role ID.

Phase 3: Sector Modules
- Tourism: Neural Story Weaver (Veo 3.1) folklore synthesis.
- Health: Encrypted Medical Vault and telemedicine links.
- Wallet: Financial settlement core using Paystack integration.
- Transport: Route synchronization via Google Maps grounding.

4. SECURITY & PUBLIC REPO SAFETY
--------------------------------------------------------------------------------
- API KEYS: Processed via `process.env.API_KEY`. No keys are hardcoded.
- LOGIN CREDENTIALS: Redacted. Admin logic uses specific email patterns 
  (e.g., [REDACTED]@iseyaa.gov.ng) for protocol elevation.
- DATA PRIVACY: All medical and financial records are simulated "ledger 
  objects" to ensure no real PII is transmitted in the demo/public node.
